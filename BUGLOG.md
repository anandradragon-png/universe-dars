## 2026-06-06 (вечер)

### Книга ДАРов грузилась на русском при EN/ES
- **Симптом:** в табе «Книги» обвязка переведена («BOOK OF DARs», «94 chapters»), но содержимое глав и «Издание 1» оставались на русском.
- **Причина:** `book-reader.js` всегда тянул `/book-chapters.json?v=1` (русский файл), хотя есть авторские оригиналы EN/ES.
- **Сделано:** оригиналы `book-chapters.en/es.json` скопированы из `public/preview/` в корень `public/`. Ридер теперь выбирает файл по языку (`_bookUrl()`), перезагружает книгу на лету при смене языка (`i18n:changed`). Токен кэша на загрузке книги.
- **Статус:** ✅ Исправлено.

### Имена и архетипы родных не переводились (Семья, Оракул)
- **Симптом:** при EN/ES в карточке «Семья» и в модалке Оракула для близких имя дара показывалось кириллицей (ТО-РА выглядит как «TO-PA»), а архетип — по-русски («Ветер перемен»).
- **Причина:** `renderRelativesBlock` и `renderRelativeOracleResult` брали имя/архетип напрямую из RU-словарей `window.DARS` / `window.DAR_ARCHETYPES`, минуя хелперы `getDarName`/`getDarArchetype`.
- **Сделано:** имя/архетип для показа берутся через `getDarName`/`getDarArchetype` (язык из i18n). Для SVG-иконки сохранён RU-вариант имени (файлы названы по-русски). Карточки перерисовываются на смену языка.
- **Статус:** ✅ Исправлено.

### Обвязка модалки Оракула для близких на русском

### ОТКРЫТО: серверный AI-текст приходит на русском при EN/ES
- **Симптом:** даже при EN/ES интерфейсе по-русски остаются: текст послания Оракула («Сегодня твоя дочь Ангелина...»), «Полное послание», главы детской «Книги для родителей» (и её разделы/возрастные метки/архетип внутри).

- **Статус:** ⏳ Открыто. Нужна серверная доработка: язык в промпт + перевод статических списков (`SECTIONS`, возрастные метки, архетипы) в `child-book.js`. Отдельная задача, ждёт приоритета Светы.

- **Статус:** ⏳ Открыто.

## 2026-06-06

### Полная локализация всего приложения (RU/EN/ES) — JS-рендеримый контент
- **Симптом:** при EN/ES интерфейсе целые модули, рендеримые из JS, оставались на русском: Сокровищница (теневой квест, диалог наставника), Путешествие Героя (шаги, битва с Тенью, итоги), Интуиция (режимы, рейтинг магов, правила), Легенды Атлантиды (игровой лог, правила, конец партии), Дар дня (вкладки, апселл, модалка апгрейда), Стендап- и подробная карточки, Ридер книги (настройки, темы, поиск, закладки), модуль АРКА.
- **Причина:** строки рендерятся в JS (innerHTML/canvas/тосты), минуя `applyToDOM`, поэтому захардкоженный русский не переводился.
- **Сделано:** 8 JS-файлов обёрнуты в `i18n.t()` с русским фоллбэком (≈466 ключей × 3 языка добавлено в `i18n/ru|en|es.json`). АРКА переведена в собственных словарях `ARKA_I18N` (187 ключей) и `ARKA_DYN_I18N` (164). Отдельно исправлена интерполяция гендерных слов в Сокровищнице: формы `{pass}/{act}/{opened}/{felt}/{ready}` подставлялись русскими словами в EN/ES-шаблоны → вынесены в i18n-ключи `gw_*` (RU сохраняет род, EN/ES нейтральны).
- **Статус:** ✅ Исправлено. Проверено в превью: EN/ES резолвят все новые ключи, гендер-шаблоны интерполируются без кириллицы и без остаточных `{...}`. Паритет ключей ru=en=es по всем namespace. `node --check` по всем 9 JS прошёл.

### Уведомления (toasts/alerts/ошибки) выводились на русском при EN/ES
- **Симптом:** заголовки и кнопки переводились, но всплывающие уведомления и ошибки сети оставались на русском: ошибки API (`api-client.friendlyError`), тосты в Сокровищнице (открыт новый дар / все дары открыты / не удалось открыть / сначала рассчитай дар), Путешествие Героя («Напиши хотя бы пару предложений»), Интуиция («+1 попытка»), Стендап-карточка (сначала сгенерируй / карточка сохранена / не удалось скачать), Подробная карточка.
- **Причина:** строки тостов/алертов рендерятся в JS захардкоженным русским, минуя i18n. `applyToDOM` их не трогает (это не DOM-элементы с data-i18n, а аргументы showToast/alert).
- **Сделано:** добавлены секции `errors.*` и `toast.*` + ключи `treasury.dar_unlocked_toast/dar_unlocked_alert/all_dars_open/unlock_random_failed/calc_first` в ru/en/es.json. Все уведомления обёрнуты в `i18n.t(key, vars) || fallback` (с `{name}`/`{msg}`-интерполяцией где нужно). Файлы: `api-client.js`, `index.html`, `treasury.js`, `hero-journey.js`, `intuition-game.js`, `humor-card.js`, `share-card.js`. JSON и JS прошли проверку синтаксиса; ключи отдаются EN из `/i18n/en.json`.
- **Статус:** ✅ Исправлено.

### ARKA: динамические строки не переключали язык
- **Симптом:** статичный HTML ARKA переводился (заголовки), а строки из JS (типы задач, советы наставника) оставались на русском при EN/ES.
- **Причина:** `dynLang()` в `preview/arka/app.js` читал только ключ `_yupdar_preview_lang`, а прод-приложение хранит язык в `_yupdar_lang`. Статичный `getArkaLang()` был починен (читает оба ключа, баг Helga 02.06.2026), а динамический — нет → `dt()` всегда падал на `ru`.
- **Сделано:** `dynLang()` теперь читает оба ключа (как `getArkaLang`). Добавлен ре-рендер динамики на смену языка. CSS-плейсхолдер задачи переведён через `attr(data-ph)`. Проверено в превью: `dynLang()='en'`, типы задач и плейсхолдер на EN.
- **Статус:** ✅ Исправлено.

### Кэш Telegram: фиксы и переводы «слетают»
- **Симптом:** починенные баги возвращались, переводы EN/ES не подтягивались в приложении и на сайте.
- **Причина:** Telegram webview кэшировал старые JS/CSS/словари, игнорируя `Cache-Control: no-cache`. Версии файлов (`?v=1`, `?v=2`…) бились вручную и часто забывались (напр. `share-card.js?v=1` не менялся с этапа 6, хотя файл правили дважды).
- **Сделано:** единый токен `?v=20260605` для всех JS/CSS и для загрузки словарей в `i18n.js`. Закреплено законом `law_cache_busting.md` — бампить на каждом деплое.
- **Статус:** ✅ Исправлено. Подтверждено: после обновления у Светы загрузился EN-интерфейс.
- **Коммит:** `6a79ae1`
