Документ для передачи проекта новому разработчику.

## 1. Что такое YupDar

YupDar — это Telegram Mini App (веб-приложение внутри Telegram) вокруг авторской системы «81 Дар». Каждый человек по дате, времени и месту рождения получает свой персональный Дар — код из трёх полей (МА — магия, ЖИ — жизнь, КУН — мощность). Приложение рассчитывает Дар, выдаёт персональные AI-послания, ведёт игры, энциклопедию Даров, книги и личный кабинет.

YupDar — входное окно (витрина) всей «Вселенной ДАРов»: зонтичного бренда автора (Светланы). Под него планируется собрать все продукты сообщества YupLand.

Адреса:
- Боевой сайт: https://yupdar.com и https://yupdar.yupland.io (один и тот же сервер)
- Бот: @YupDarBot
- Веб-зеркало (старое, Vercel): public-yup-land1.vercel.app

## 3. Хостинг и деплой (важно для разработчика)

Раньше проект жил на Vercel. С 13 июня 2026 полностью переехал на собственный сервер сообщества YupLand.

- Сервер: 151.243.217.138 (YupLand)
- Запуск: Docker, образ yupdar:latest, контейнер слушает порт 3000, наружу проброшен 3001
- Обратный прокси: общий Caddy на сервере (он выпускает SSL Let's Encrypt и проксирует yupdar.com, www.yupdar.com и yupdar.yupland.io на контейнер). Конфиг Caddy ведёт администратор сервера (Сэляви), не из этого репозитория.
- DNS: Cloudflare, режим DNS only (proxied: false), A-записи yupdar.com и www → 151.243.217.138
- Автодеплой: GitHub Actions (.github/workflows/deploy.yml). На каждый push в main — заход по SSH в /opt/yupdar, git pull, docker compose build --no-cache и up -d. Время деплоя ~1 минута.
- Секреты GitHub Actions: SSH_HOST, SSH_USER, SSH_PORT (2222), SSH_KEY.
- Переменные окружения приложения лежат на сервере в файле .env.production (НЕ в репозитории).

Репозиторий: github.com/anandradragon-png/universe-dars, основная ветка main.

Важно про домен: yupdar.com зарегистрирован через Vercel как регистратора. Перенос регистратора заблокирован правилами ICANN до 24 июля 2026 (60 дней с момента покупки). Аккаунт Vercel был заморожен из-за долга $20 за домен.

Файлы деплоя в репозитории: Dockerfile, docker-compose.yupdar.yml, server.js.

## 8. Реализованные продукты и фичи

Расчёт Дара. По дате/времени/месту рождения вычисляется код из 3 полей. Личные данные (дата, время, место, имя, пол) — read-only везде, изменение только через поддержку (защита от абуза).

Энциклопедия Даров. 64 обычных Дара + 17 интегралов = 81. Каждый Дар: 9 секций контента. Источник — dar-content.json (НЕПРИКОСНОВЕНЕН, меняется только с согласия автора).

Полное послание. Главная AI-функция: персональный текст о Даре. Промт заморожен (финал v7), эталон авторского стиля вшит в промт. DeepSeek + Groq fallback.

Стендап-зеркало. Ироничный AI-портрет по Дару, с опорой на поля МА/ЖИ/КУН и тени. Тёплая ирония, без эзотерики и инженерных метафор.

Оракул. Тарологическая механика: пользователь задаёт вопрос → выпадает 1 из 81 Дара → AI трактует вопрос через выпавший Дар, с учётом личного Дара пользователя. 8 готовых вопросов + своё поле.

Совместимость и родственники. Анализ совместимости двух Даров, послания для родственников (от 3-го лица).

Игры: «Интуиция» (с рейтингом и наградами), «Легенды Атлантиды» (карты, параметры Магия/Жизнь/Сила/Мощность), Путь героя, квесты.

Сокровищница и кристаллы. Внутренняя валюта. Сейчас большинство наград заморожено (FROZEN_REWARDS), активны только награды игры Интуиции и реальные покупки — до появления внятной полезности трат кристаллов.

Рефералы и шеринг. Приглашения друзей, виральные карточки.

Книги. Читалка трёх книг (RU/EN/ES): «Книга Даров» и материалы. Темы оформления, поиск, закладки, lightbox для картинок.

Личный кабинет. Профиль, тариф, история.

Локализация. RU / EN / ES. Языковой переключатель ВСЕГДА в шапке (закон — нельзя прятать в меню/ЛК).

Платежи. ЮKassa (рубли), Telegram Stars, T-Bank.

Админ-панель. Аналитика, статистика, управление пользователями и тарифами, промокоды, баланс Stars, очистка кэша.

АРКА (в превью, ещё не в основном проде). Личный планировщик «в ритме Дара» — премиум-продукт внутри YupDar. Живёт в public/preview/arka/. Реализованы: тема «Сумерки времени», 4 эксклюзивных раздела в табе «Дар» (Окна силы, Карта роста, Дневник эволюции, Тренажёр), Зеркало момента с Даром дня, шеринг-карточки, Открытие тени, Письмо на 30 дней, Кольцо силы.

Дневник. Бесплатный ежедневный крючок (эмодзи + мини-зеркало с CTA на АРКА). Живёт в public/preview/diary/.

## 10. Кэш-бастинг (обязательная процедура на каждый деплой)

Telegram webview агрессивно кэширует старый JS/CSS/словари — без этого фиксы и переводы «слетают» у пользователей.

Правило: единый токен ?v=ГГГГММДД(буква) у ВСЕХ ассетов в index.html + дублируется в public/version.txt. Бампить на КАЖДОМ деплое, который меняет фронтенд. Текущее значение: 20260622a.

## 12. Законы и правила проекта (критично соблюдать новому разработчику)

Эти правила накапливались месяцами и нарушение каждого однажды ломало продукт:

- Сервер — источник правды, localStorage — только кэш. Никогда не показывать пустой экран, если Дар уже рассчитан (восстановление с сервера при каждом запуске, 3-уровневый fallback).
- Архитектура заморожена (25 мая 2026): 4 таба — Я / Семья / Сокровищница / АРКА. Карточка Дара + одна кнопка «Читать о Даре» → overlay с аккордеоном + шеринг внизу.
- Поток «Полное послание» зафиксирован: кнопка «Получить полное послание» → AI → меняется на «Читать о своём Даре» → overlay. Не переусложнять.
- dar-content.json и тексты книг — неприкосновенны, меняются только с согласия автора.
- Все тексты на русском, без латиницы, без длинного тире (—), простой язык. Русская версия — только русский, никаких ключей i18n или английских слов.
- Языковой переключатель — всегда в шапке.
- Тарифная иерархия заморожена: Странник < Хранитель < Мастер (Мастер высший).
- Стиль AI-генерации: только «ты», «важно» вместо «нужно», без «человек/он/Вы», без повторов. Вся персональная генерация опирается на конкретный Дар, никакого общего «мудрого» контента.
- Личные данные read-only везде.
- Минимум на экране: всё что можно свернуть — свёрнуто, разворачивается по клику. Не перегружать UI.
- Названия кнопок/заголовков/секций менять только с разрешения автора.
- Юр.документы ЮKassa (public/legal/ru, /en, /es) не трогать — идёт проверка.
- Деплои собирать в пачки (один логический блок = один деплой), без пустых коммитов.

## 15. Общее видение развития

YupDar задуман не как одно приложение, а как главная дверь во «Вселенную ДАРов» — экосистему вокруг авторской системы 81 Дара. Логика развития:

Сейчас (ядро). Бесплатный вход через расчёт Дара и AI-послания → знакомство с системой. Игры и Дневник — ежедневные крючки удержания. Энциклопедия и книги — глубина.

Следующий слой (монетизация). Платное ядро — тарифы Хранитель и Мастер, разовая покупка Книги. Главный платный продукт — АРКА: личный планировщик в ритме Дара, который превращает знание о своём Даре в ежедневную практику (целевые действия, ритуалы, масла, тень, эволюция).

Масштаб (экосистема). YupDar становится витриной для всех сервисов сообщества YupLand: Паспорт Дара как единый идентификатор, Академия, другие продукты. Локализация RU/EN/ES открывает международный рынок (Amazon KDP для книг, Paddle для зарубежных платежей).

Принципы продукта, которые нельзя терять при развитии: персонализация всегда строится на конкретном Даре (а не общих советах); минимализм интерфейса (показывать только нужное сейчас); живой авторский голос в текстах (не «машинный»); язык утверждений без частицы «не»; уважение к личным данным.
