### Серверный AI-текст приходил на русском при EN/ES
- **Симптом:** даже при EN/ES по-русски оставались: послание Оракула («Сегодня твоя дочь Ангелина...»), «Полное послание», главы детской «Книги для родителей», контент «Сегодня» в АРКА.
- **Причина (несколько слоёв):**
  1. Префикс `applyLanguage` (EN/ES-инструкция) добавлялся сверху, но в ТЕЛЕ промптов оставались строки «Только русский / Никакой латиницы / Не пиши иностранных слов» — LLM мог послушаться их.
  2. `content.js` → `sanitizeOracleOutput`/`spellCheckOracleOutput` вырезали ВСЮ латиницу (`replace(/[a-zA-Z]+/g,'')`) и гнали русский Яндекс-спеллер → уничтожали английский/испанский текст.
  3. `message.js` → `russifyText` (строка 95) стирала любое латинское слово целиком + `feminizeText` — тоже фатально для EN/ES.
  4. Кэш Оракула (`oracle_cache`) и детской книги (`child_book_sections`) не учитывал язык → русский кэш за день перебивал EN/ES.
- **Решение Светы (06.06.2026):** правило «только русский» в промптах действует ТОЛЬКО когда выбран русский. При EN — весь ответ на английском, при ES — на испанском, без примеси других языков. Это снимает блокировку с замороженного промпта В РАМКАХ языка (тело промптов НЕ менялось — закон `law_message_prompt_final.md` соблюдён).
- **Сделано:**
  - `language.js`: в EN/ES-инструкцию добавлен жёсткий OVERRIDE — явно отменяет правила «только русский/без латиницы» из тела промпта для не-русского языка. Чинит сразу ВСЕ промпты (Оракул, Полное послание, АРКА, Совместимость, Детская книга).
  - `content.js` (Оракул): `sanitizeOracleOutput`/`spellCheckOracleOutput` получили `lang` и пропускают вырезание латиницы + русский спелчек для не-ru. Ключ кэша теперь с суффиксом языка (`date_key` → `2026-06-06:en`) — без миграции БД (для ru ключ прежний, обратная совместимость; для EN/ES при «date»-колонке кэш просто отключится → всегда свежая генерация на нужном языке).
  - `message.js` (Полное послание): `russifyText`/`feminizeText` применяются только при ru (передан `lang` в `_parseAndPostprocess`).
  - `child-book.js`: `applyLanguage` к системному промпту + ключ кэша с суффиксом языка + перевод статичного оглавления (`SECTION_I18N`) и возрастных меток (`AGE_LABEL_I18N`) для EN/ES.
  - `arka-today.js`: уже передавал `lang` → закрыт правкой `language.js`.
- **Статус:** ✅ Исправлено (серверная часть). Локально в статичном превью не тестируется (нет `/api`) — проверка на проде в Telegram. RU-путь не затронут нигде.
- **Примечание по БД:** при подключённом к Claude MCP-Supabase оказалась ДРУГАЯ база (YupLand: `app_users`, `yl_*`), не YupDar. Схему реальной БД проверить нельзя — поэтому язык вложен в существующий текстовый ключ кэша, а не отдельной колонкой.

Дебаунс через `setTimeout(0)` (не `requestAnimationFrame` — тот не тикает в фоне). Вызовы: showResult (+300/+1000мс) и `_refreshDarLabels`. Проверено в превью: текстовый узел, div-сирота, вложенный span И асинхронная вставка — все вычищены, `#darName` цел. Токен `20260606e`.
